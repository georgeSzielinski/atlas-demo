# Atlas AI Scorecard

Use this scorecard before integrating any specialized AI system into Atlas.
Scores should be evidence-backed and reproducible.

Atlas uses controlled learning. Benchmark reports may suggest tuning ideas, but
Atlas does not self-modify. Any adjustment requires human approval before code,
configuration, model, or recommendation behavior changes.

| Criterion | Score | Notes |
| --- | --- | --- |
| Accuracy | TBD | Directional accuracy, extraction quality, or task-specific correctness. |
| Recommendation Quality | TBD | BUY accuracy, HOLD accuracy, AVOID accuracy, overall hit rate, average return, average gain, and average loss. |
| Confidence Calibration | TBD | High-confidence accuracy and low-confidence accuracy. |
| Signal Quality | TBD | High-signal-quality accuracy, low-signal-quality accuracy, and sample count. |
| Forecast Quality | TBD | Direction accuracy, MAE placeholder, RMSE placeholder, and runtime placeholder. |
| Evidence Effectiveness | TBD | Effectiveness score, sample count, and last benchmark date by evidence source. |
| Speed | TBD | Local runtime, API latency, and batch throughput. |
| Memory | TBD | RAM, VRAM, and model loading footprint. |
| License | TBD | Compatibility with Atlas use and redistribution constraints. |
| Maintenance | TBD | Dependency stability, release cadence, and operational burden. |
| Community | TBD | Adoption, documentation quality, issue activity, and examples. |
| Integration Difficulty | TBD | Fit with Atlas interfaces, data contracts, and failure handling. |
| Atlas Score | TBD | Overall weighted score for Atlas suitability. |
